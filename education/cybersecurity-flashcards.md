## üéì Week 1: Cybersecurity Fundamentals

### **Day 1: Core Concepts**

| **Term** | **Definition** |
|----------|----------------|
| **Cybersecurity** | The practice of protecting digital information, systems, and networks from cyber threats |
| **CIA Triad** | Confidentiality, Integrity, and Availability - the three pillars of information security |
| **Confidentiality** | Ensuring only authorized people can access information |
| **Integrity** | Ensuring information remains accurate and unaltered |
| **Availability** | Ensuring information and systems are accessible when needed |
| **Threat** | Any potential danger to information security |
| **Vulnerability** | A weakness in a system that can be exploited |
| **Risk** | The probability of a threat exploiting a vulnerability |
| **Asset** | Anything of value that needs protection |
| **Attack Vector** | The path or means by which an attacker gains access to a system |

### **Day 2: Threat Types**

| **Term** | **Definition** |
|----------|----------------|
| **Malware** | Malicious software designed to harm, disrupt, or gain unauthorized access |
| **Virus** | Malware that replicates by attaching to other programs |
| **Worm** | Self-replicating malware that spreads across networks |
| **Trojan** | Malware disguised as legitimate software |
| **Ransomware** | Malware that encrypts data and demands payment for decryption |
| **Phishing** | Fraudulent attempt to steal sensitive information via deceptive communications |
| **Spear Phishing** | Targeted phishing attack against specific individuals or organizations |
| **Social Engineering** | Psychological manipulation to gain access to systems or information |
| **DDoS** | Distributed Denial of Service - overwhelming a system with traffic |
| **APT** | Advanced Persistent Threat - sophisticated, long-term cyber attack |

### **Day 3: Risk Management**

| **Term** | **Definition** |
|----------|----------------|
| **Risk Assessment** | The process of evaluating potential security risks |
| **Risk Management** | The process of identifying, assessing, and controlling risks |
| **Risk Tolerance** | The level of risk an organization is willing to accept |
| **Risk Mitigation** | Actions taken to reduce the likelihood or impact of risks |
| **Risk Transfer** | Shifting risk to another party (e.g., insurance) |
| **Risk Acceptance** | Acknowledging and accepting certain risks without mitigation |
| **Risk Avoidance** | Eliminating activities that create risk |
| **Business Impact Analysis** | Assessment of potential effects of business disruptions |
| **Residual Risk** | Risk remaining after security controls are implemented |
| **Inherent Risk** | Risk before any controls are applied |

### **Day 4: Access Control**

| **Term** | **Definition** |
|----------|----------------|
| **Authentication** | The process of verifying the identity of a user or system |
| **Authorization** | Granting or denying access to specific resources |
| **Accounting** | Tracking and recording user activities and resource usage |
| **Multi-Factor Authentication (MFA)** | Using multiple authentication factors for verification |
| **Something You Know** | Authentication factor based on knowledge (passwords) |
| **Something You Have** | Authentication factor based on possession (tokens, phones) |
| **Something You Are** | Authentication factor based on biometrics |
| **Single Sign-On (SSO)** | Using one set of credentials to access multiple systems |
| **Least Privilege** | Granting minimum access necessary to perform job functions |
| **Privileged Access Management (PAM)** | Managing and monitoring access to critical systems |

### **Day 5: Network Security**

| **Term** | **Definition** |
|----------|----------------|
| **Firewall** | Network security device that monitors and filters traffic |
| **Network Segmentation** | Dividing a network into smaller, isolated segments |
| **VPN** | Virtual Private Network - secure connection over public networks |
| **IDS** | Intrusion Detection System - monitors for malicious activity |
| **IPS** | Intrusion Prevention System - actively blocks malicious activity |
| **DMZ** | Demilitarized Zone - buffer network between internal and external |
| **Proxy Server** | Intermediary server that forwards requests between clients and servers |
| **Network Access Control (NAC)** | Security approach controlling device access to networks |
| **VLAN** | Virtual Local Area Network - logical network segmentation |
| **Zero Trust** | Security model requiring verification for every access request |

### **Day 6: Cryptography**

| **Term** | **Definition** |
|----------|----------------|
| **Cryptography** | The practice of secure communication through encoding |
| **Encryption** | Converting plaintext into unreadable ciphertext |
| **Decryption** | Converting ciphertext back into readable plaintext |
| **Symmetric Encryption** | Using the same key for encryption and decryption |
| **Asymmetric Encryption** | Using different keys for encryption and decryption |
| **Public Key** | Publicly shared key in asymmetric cryptography |
| **Private Key** | Secret key in asymmetric cryptography |
| **Hash Function** | One-way mathematical function that produces fixed-length output |
| **Digital Signature** | Cryptographic mechanism to verify authenticity and integrity |
| **Certificate Authority (CA)** | Trusted entity that issues digital certificates |

### **Attack Types**

| **Term** | **Definition** |
|----------|----------------|
| **Brute Force** | Attempting to guess passwords through repeated tries |
| **Dictionary Attack** | Using common passwords from a predefined list |
| **Man-in-the-Middle** | Intercepting communications between two parties |
| **Replay Attack** | Capturing and retransmitting data to gain access |
| **Session Hijacking** | Taking over a user's active session |
| **Privilege Escalation** | Gaining higher-level access than originally granted |
| **Lateral Movement** | Moving through a network after initial compromise |
| **Watering Hole** | Compromising websites frequently visited by targets |
| **Supply Chain Attack** | Compromising software or hardware before delivery |
| **Living off the Land** | Using legitimate tools for malicious purposes |
